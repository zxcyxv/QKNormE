# QKNormGaussian 연구 위키 스키마

## 개요
QK-Norm 조건 하에서의 트랜스포머 어텐션 동역학에 관한 딥러닝 연구 위키.
LLM이 생성·유지관리하며, 모든 위키 페이지는 한국어로 작성한다.

## 디렉토리 구조

```
raw/                  # 원본 자료 (불변, LLM이 수정하지 않음)
  assets/             # 이미지 등 첨부파일
wiki/                 # LLM이 관리하는 위키 페이지
  index.md            # 전체 페이지 카탈로그
  log.md              # 작업 이력 (append-only)
  overview.md         # 연구 전체 개관
  synthesis.md        # 자료 간 종합 분석
  sources/            # 원본 자료별 요약 페이지
  concepts/           # 개념 페이지 (수학적 개념, 기법 등)
  entities/           # 엔티티 페이지 (저자, 모델, 논문 등)
  reports/            # ChatGPT 보고서 (자동 저장)
report.py             # ChatGPT 보고서 생성 스크립트
```

## 페이지 형식

모든 위키 페이지는 YAML frontmatter를 포함한다:

```yaml
---
title: 페이지 제목
tags: [태그1, 태그2]
sources: [관련 원본 자료 파일명]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

본문에서 다른 위키 페이지를 참조할 때는 Obsidian 스타일 위키링크를 사용한다: `[[페이지명]]`

## 워크플로우

### Ingest (자료 투입)
1. 원본 자료를 `raw/`에 배치
2. 자료를 읽고 핵심 내용을 사용자와 논의
3. `wiki/sources/`에 요약 페이지 작성
4. 관련 개념·엔티티 페이지를 생성하거나 업데이트
5. `wiki/index.md` 업데이트
6. `wiki/log.md`에 작업 기록 추가

### Query (질의)
1. `wiki/index.md`를 읽어 관련 페이지 탐색
2. 관련 페이지들을 읽고 종합하여 답변
3. 가치 있는 답변은 새 위키 페이지로 저장

### Report (외부 보고서 생성 — ChatGPT 연동)

ChatGPT에 연구 질문을 보내 보고서를 받아오고, 검증 후 위키에 통합하는 워크플로우.

**절차:**
1. 사용자가 연구 질문을 요청한다
2. Claude가 질문 의도를 파악하고, 위키/소스에서 관련 자료를 선별하여 첨부 준비한다
   - 소스가 길면 핵심만 요약하여 첨부, 짧으면 원문 그대로 첨부
3. Claude가 정교하게 다듬은 프롬프트 초안을 사용자에게 제시한다
   - 첨부할 소스 목록 + 프롬프트 본문을 보여주고 컨펌을 받는다
4. 사용자 컨펌 후 `report.py`를 통해 ChatGPT에 전송한다
   ```bash
   uv run python report.py -p "프롬프트" -s raw/소스1.pdf raw/소스2.txt
   ```
5. 보고서가 `wiki/reports/`에 자동 저장된다
6. Claude와 사용자가 보고서를 함께 검토한다
   - 맞는 내용 / 틀린 내용 / 추가 논의 필요 사항을 구분한다
7. 검증된 내용을 기존 위키 페이지에 통합하거나 새 페이지로 작성한다
8. `wiki/log.md`에 기록한다

**사용 스크립트:** `report.py`
- `--prompt, -p`: 프롬프트 (문자열 또는 .txt 파일 경로)
- `--sources, -s`: 첨부할 소스 파일 경로들
- `--max-pages`: PDF 최대 페이지 수 제한
- `--model, -m`: 모델 오버라이드 (기본: gpt-5.4-pro, 환경변수 OPENAI_MODEL)
- `--no-save`: 파일 저장 생략
- 환경변수 `OPENAI_API_KEY` 필요

### Lint (점검)
1. 페이지 간 모순 확인
2. 고아 페이지(인바운드 링크 없음) 확인
3. 누락된 교차 참조 보완
4. 새 소스로 인해 갱신이 필요한 페이지 식별

## 규칙
- `raw/` 파일은 절대 수정하지 않는다
- 위키 페이지 작성 시 수학 수식은 LaTeX(`$...$`, `$$...$$`)를 사용한다
- 주장에는 반드시 출처를 명시한다
- 서로 다른 자료 간 모순이 있으면 명시적으로 기록한다
